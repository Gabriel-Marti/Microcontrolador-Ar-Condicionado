Inicialmente um sensor receptor de infraveremelho, juntamente ao código de receptor IR, código de exemplo padrão, "IRrecvDumpV3", da biblioteca usada, "IRremoteESP8266", esse código disponível por padrão na biblioteca foi feito para identificar o protocolo de emissão de infravermelho, ao disparar o sinal no receptor adequadomente, que já está previamente regitrado na biblioteca, assim ao descubrir o protocolo de IR do dispositivo em questão, basta trocar o protocolo no código e o controlador se addequará ao dispositivo imediatamente

Para a conexão de rede, é necessário substituir as variáveis de coenctividade, referente aos dados de conexão do ambiente em que o sistema será instalado, a conexão é feita por wifi, pelo protocolo mqtt através de um broker apropriado, a ideia é que a conexão seja feita pela própria rede interna. A ideia inciial é fazer uma conexão simples sem criptografia ou certificados, uma vez que os dados compartilhados não devem ser sensíveis não há necessidade de adicionar complexidades desnecessárias

Todos os sistemas embarcados em questão estarão distribuídos de modo a se conectar a um servidor central, capaz de gerenciar todos eles, foi construído um protótipo de site web como exemplo, com foco no paradigma orientado a objetos, cada embarcado que se conecta a um servidor gera um novo objeto, exibido na interface, esse objeto possui dados como temperatura, umidade, estado do ar condcionado, identificador do aparelho. 
Com uma pesquisa de amostra de público alvo, foi notável que Um problema existente é referente à dificuldade em se ter um registro de manutenção para os dispositivos. Assim, uma solução seria fazer cada objeto (instanciado da classe que contém o script referente à representação do microcontrolador no servidor) se conectar a uma api de calendário e deixar o registro de manutenção em algum banco de dados que tenha backup, para que o registro não seja deletado caso o embarcado desapareça, de modo os objetos não armazenem esses dados, somente os exibam. Entretanto, deve ser possível editar os dados de registro a partir do servidor central

O protótipo do site teve seu código escrito por inteligência artificial, mas a arquitetura e a lógica principal dele foi projetada pelo grupo
